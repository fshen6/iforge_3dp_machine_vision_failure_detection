## What this is

Greenfield ML system. Test plan is forward-looking: tests to write as the code lands, not tests against existing code.

## Affected components (none exist yet)

- `octoprint_client.py` — REST polling + snapshot pull + pause API
- `inference_service.py` — frame capture loop, stage1+stage2 prediction, rolling-window decision
- `action_layer.py` — OctoPrint pause call with retry, ntfy notification with action buttons
- `labeling/feedback.py` — handle ntfy button-tap webhook, write operator label to SQLite
- `cron/retain.py` — 90-day frame retention cleanup
- `eval/grade.py` — score a model against the curated eval set

## Key interactions to verify

### Print-start detection
- Service polls `/api/job` every 5s, transitions Operational → Printing trigger a monitoring window open
- Multiple print-starts (all 10 printers at once) handled concurrently, not serialized

### Inference pipeline (shadow mode)
- Pull snapshot from OctoPrint → run stage 1 → if failure, run stage 2 → write to SQLite
- Rolling-window vote: 3-of-5 frames flagged at stage 1 → would-pause decision
- In shadow mode, `pause_print()` is replaced with `log_what_would_have_happened()`

### Pause + notify (live mode)
- Stage 1 fires 3-of-5, action layer calls OctoPrint pause
- Retry: if pause API returns non-2xx, retry at 2s/5s/10s
- All three retries fail → "PAUSE FAILED, GO LOOK" notification with high priority
- Pause success → standard notification with snapshot, predicted class, two action buttons

### Labeling feedback
- Operator taps `Real failure` → ntfy webhook → SQLite row updated with `operator_label='confirmed'`
- Operator taps `False alarm` → SQLite row updated with `operator_label='false_positive'`
- Operator doesn't tap → row stays `operator_label=null`, system doesn't break

## Edge cases (must have tests)

- **Camera 404 / blank frame** — service should skip the frame, increment a counter, alert if >5 consecutive failures
- **OctoPrint API timeout** during pause — retry with backoff, eventually escalate
- **Inference service crash** mid-print — systemd restarts, service reads SQLite to know which printers were being monitored, resumes windows
- **Power loss** on mini-PC — systemd starts service on boot, state recovers from SQLite
- **10 concurrent print-starts** — service handles parallel monitoring windows correctly, no race on SQLite writes
- **Phone offline** when notification fires — ntfy queues, delivers on reconnect (manual smoke test only)
- **Operator taps a button on a stale notification** (print already ended) — webhook handler treats stale tap gracefully, still writes label
- **Print ends during a monitoring window** — service closes window cleanly, marks `prints.ended_at` and `end_reason`
- **Model returns weird confidence values** (NaN, inf, negative) — guard with sanity-check, never trigger pause on garbage

## Critical paths (end-to-end)

1. **Healthy 12-hour print runs without false pause.** Most important E2E test. Run inference on a known-good timelapse, expect 0 pause decisions.
2. **Bed-adhesion failure caught and paused.** Replay a known bed-adhesion timelapse, expect pause decision within 60s of visual onset.
3. **Mid-print spaghetti at hour 6 caught and paused.** Replay a known spaghetti timelapse, expect pause decision within 60s of onset.
4. **Shadow mode never pauses.** Run all of the above in shadow mode, expect zero OctoPrint pause calls.
5. **Operator labels the frame.** Trigger a pause, simulate ntfy button tap, verify SQLite row has correct operator_label.

## Eval (model quality)

Maintained separately from unit/integration tests:

- **Curated eval set** — 50–100 frames per failure scenario, never used for training, never used for hyperparameter tuning. Lock and version it.
- **v1 vs v2 comparison** — before promoting any new model, score it against the eval set and compare per-class precision/recall/F1 vs the current production model. v2 must beat v1 on every class to ship.
- **Time-to-detection metric** — for each timelapse in the eval set, measure how many seconds after the visual onset of failure the model first flags it. Aim: median < 60s.
- **False-positive rate on healthy prints** — score the model against the curated healthy subset, target < 1 false-pause-equivalent per printer per week at deployed thresholds.
